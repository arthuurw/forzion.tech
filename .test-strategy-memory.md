## Stack & Ferramental

**Backend:** xUnit + Moq + FluentAssertions + WebApplicationFactory + Testcontainers.PostgreSql  
**Frontend:** Vitest 4 + happy-dom + @testing-library/react + @vitest/coverage-v8  
**Coverage backend:** Coverlet (collector local + msbuild CI). Runsettings: `forzion.tech.Tests/coverage.runsettings`. Thresholds CI: 50% global + 75% branch Domain + 75% branch Application.  
**Coverage frontend:** `vitest.config.mts` → thresholds branches 65%, functions/lines/statements 70%. Script `test:coverage`.  
**Mutation testing:** `dotnet-stryker` em `dotnet-tools.json`. Config: `stryker-config.json` (target Domain, break=0, low=60, high=80). Rodar: `dotnet stryker`. Relatório em `mutation-report/`.  
**CI:** `.github/workflows/ci.yml` — ambos rodam com coverage em PR e push.

## Convenções do projeto

- Naming: `Método_Cenário_Resultado` (ex: `HandleAsync_VinculoValido_Aprova`)
- AAA implícito — sem comentários // Arrange // Act // Assert
- Mocks: Moq com `It.IsAny<CancellationToken>()` em todos os setups
- Handlers mockados em endpoint tests via `RemoveAll + AddSingleton`
- Auth em endpoint tests: `TestAuthHandler` — tokens simples `"admin"` / `"treinador"`
- `TreinadorId` injetado via reflection quando necessário
- `ArgumentNullException.ThrowIfNull(command)` em todo handler
- `HandleAsync` deve ser `virtual` em todos os handlers
- **Assertions preferidas**: estado da entidade resultante. `Times.Exactly(N)` em UnitOfWork = antipattern. `Times.Once/Never` OK para side effects explícitos (ex: Stripe não chamado).
- Namespace armadilha: `Conta` colide com entity → alias `DomainConta`; `Email` colide com VO → alias `EmailVO`
- `IUserContext.IsSystemAdmin` é default interface method — mockar explicitamente sempre

## Testcontainers — Setup

```csharp
[Collection(InfrastructureTestCollection.Name)]
public class MeuRepositoryTests(InfrastructureTestFixture fixture)
{
    [Fact]
    public async Task Meu_Teste()
    {
        await using var ctx = fixture.CreateContext();
        // usa ctx.VinculosTreinadorAluno, ctx.Alunos, etc.
        // ctx é AppDbContext real contra Postgres
        await ctx.SaveChangesAsync();
        var repo = new MeuRepository(ctx);
        ...
    }
}
```

- Fixture em `Tests/Infrastructure/InfrastructureTestFixture.cs`
- Container iniciado 1x por collection via `IAsyncLifetime`
- `EnsureCreatedAsync` (não migrations) — mais rápido, suficiente para testes
- Cada teste usa `fixture.CreateContext()` → contexto isolado
- Seed com `Guid.NewGuid():N` no nome → evita colisões entre testes paralelos
